office-causal

Turn all the XML inside MS Office (OOXML: pptx / xlsx / docx) into a single typed causal graph in TypeScript.

▶ Live WebGPU demo GitHub repo README 日本語

What it does

An Office document is a finished artifact for humans, but it hides a large amount of implicit causal structure: a spreadsheet cell B2 = A2*1.1 depends on A2; a slide claims a share decline was caused by a competitor's price cut. office-causal attaches a stable data-id and meta to every element and lifts the document from structure → reference/dependency → causal graph.

ingest → identify (data-id+meta) → structural (contains / references / derives-from) → embed (local model) → semantic → causal (direction / polarity) → verify

The deterministic layers are pure functions (no LLM); only meaning and causal adjudication use a model — keeping results reproducible, cheap and auditable.

Typed causal graph

One directed graph: edges = contains / references / derives-from / mentions / causes. Every causes edge carries evidence + a mechanism and passes hypothesis → supported/refuted.

Stable data-id

Content-addressed ids survive Office re-saves. Embed the whole graph back into the file non-destructively as .ocz.pptx — it stays valid OOXML and opens normally.

Local / private

Cloud-free edge-LLM: encoder embeddings → local Gemma 4 (WebGPU) adjudicates direction/polarity and runs the adversarial verify. No API key, no data leaves the browser.

Diagnose & consult

Mechanically find isolated nodes, refuted edges, notation variants and concept jumps; walk causal chains for so-what implications + MECE checks.

Quick start

Try it now — no install

Open the live WebGPU demo in Chrome/Edge, drop a .xlsx/.pptx/.docx (or use the sample text), then Run → 🔍 diagnose → 💡 so-what/MECE. Everything runs in the browser.

Use it as a library

# configure the scope registry + auth in your project .npmrc
echo "@com-junkawasaki:registry=https://npm.pkg.github.com" >> .npmrc
echo "//npm.pkg.github.com/:_authToken=${GITHUB_TOKEN}" >> .npmrc   # PAT with read:packages

npm install @com-junkawasaki/office-causal
import { analyze, embedFile, readDataPart, diagnose, consult } from "@com-junkawasaki/office-causal";

const graph = await analyze("report.pptx", { llm: { device: "cpu", verify: false } });
await embedFile("report.xlsx", { mode: "part" });          // → report.ocz.xlsx
const data = readDataPart(new Uint8Array(fs.readFileSync("report.ocz.xlsx")));

Use the CLI

office-causal analyze  report.pptx                  # → CausalGraph (JSON)
office-causal embed    report.pptx --analysis      # → report.ocz.pptx (graph + analysis embedded)
office-causal diagnose report.ocz.pptx --gemma     # isolated / not-holding / notation / concept-jumps
office-causal consult  report.ocz.pptx --mece --svg sowhat.svg
office-causal locate   report.ocz.pptx <data-id>   # deep-link an element

Output examples — diagnosis SVG

diagnose --svg out.svg renders a self-contained annotated SVG. Layout adapts to the source: docx/text uses a topological-DAG layout, xlsx places cells on the Sheet!A1 grid, pptx reproduces the slide using shape bbox. Edges: green = holds, red dashed = not-holding; +/- = polarity; gray box = isolated.

Design decisions (ADR)

  • ADR-0001 — single CausalGraph IR / deterministic-vs-LLM split / content-addressed data-id / evidence-required causal edges
  • ADR-0002 — transformers.js local embeddings for edge weights & tagging
  • ADR-0003 — edge-LLM tiering (embedding → local Gemma 4/WebGPU adjudication + verify), cloud-free
  • ADR-0004 — non-destructive OOXML embedding (OPC part / attrs / bookmarks / JSONL / diff / deep-link)
